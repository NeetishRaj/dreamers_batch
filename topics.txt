Part 9: Routing in React
########################


-    In routing our basic task as React developers is just to connect
    the routes with their coresponding components.

-   Reference for React Router 
        -https://reactrouter.com/en/main


-   Nested Routes
        Example: /contact/nitin

-   Query Parameters
        In any URL anythin after '?' is considered a query parameter.
        They are separated by '&'


        Example:-
            https://abc.com?<query params>

            https://abc.com?id=1234&token=aksfjwerwer23452
            Here 'id' and 'token' are query parameters


-   Route Parameters
    
        In Route parameter the varaible is within '/' 

        https://abc.com/accounts/{route paramter}

        Reference: https://reactrouter.com/en/main/hooks/use-params

        Examples:-

            https://abc.com/accounts/2345
                This will open profile for person with id 2345
                
            https://abc.com/accounts/5678
                This will open profile for person with id 5678








Important Notes
###############

Q. How to add routing to my existing react project?

A. These are the steps

    1. Install "react-router-dom" package.
        npm install react-router-dom

    2. Wrap the complete <App> component with BrowserRouter component
        import { BrowserRouter } from "react-router-dom";

        <BrowserRouter>
            <App />
        </BrowserRouter>

    3. Specify all the routes and connect them with their corresponding components

        <Routes>
            <Route path="/">
                <Route index element={<HomePage />} />

                <Route path="about" element={<AboutUsPage />} />

                <Route path="*" element={<PageNotFound />}/>
            </Route>
        </Routes> 

    4. [Optional]: Specify the outlet where the routed components will be rendered

        <Outlet />

    5. Now we can use the <Link> to create links to our React routes

        import { Link } from "react-router-dom";

        <nav>
            <Link to="/"><div>Home</div></Link>
            <Link to="/about"><div>About Us</div></Link>
            <Link to="/contact"><div>Contact Us</div></Link>
        </nav>


Q. What's the difference between Route parameters and Query parameters?

A. 
    Route Parameters
        - It comes before '?' and between the '/' 

    Query Parameters
        - It comes after the '?' and separated by '&'


    Example with both route & query params:-
        https://bank.com/{region}/person/{pesonId}?loggedin=yes&id=5678
        Here 
            region & personId   -> Route parameters
            loggedin & id       -> Query Parameters
